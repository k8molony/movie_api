Welcome to the documentation page!

The purpose of this project is to build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request URL HTTP Method Request data format Response data format
Get a list of all movies /movies GET None A JSON object holding data about all the movies
Get data about a single movie by title /movies/[Title] GET None A JSON object holding data about a single movie
Example:
{
id: 1,
title: "Captain America: The First Avenger",
description: "Origin story of Captain America and the beginning of all of the Avengers",
setting: 1942,
releaseDate: 2011,
director: Joe Johnston,
image url: "www.link.to/movie/coverartwork"
}
Get data about setting or release date by year /movies/?setting=[setting] or /movies/?releaseDate=[releaseDate] GET The search query should contain "setting" or "releaseDate" A JSON object holding all movies assigned to a specific setting or release date.
Get data about a director /directors/[Name] GET None A JSON object holding data about a director containing bio, birth year and death year.
Example:
{
bio: "Joe Johnston is an American film director known for his special effects.",
birthYear: 1950,
deathYear: N/A
}
Allow new users to register /users POST A JSON object holding data containing user name, first name, last name, and email
Example:
{
userName: "CaptainSteve",
firstName: "Steve",
lastName: "Rogers",
email: "captainsteve@gmail.com"
} 		A verification message if the registration has been successful or not.
Allow users to update their user data by user name /users/:userName PUT A JSON object holding data containing the user data to be changed.
Example:
{
firstName: "Sam",
lastName: "Wilson",
} 		A verification message that a new username was successfully assigned or failed.
Allow users to add a movie to their list of favorites /users/:userName/favorites PUT A JSON object holding the name of the movie to be added.
Example:
{
name: "movieName",
} 		A message whether the film has been added successfully to favorites or not.
Allow users to remove a movie from their list of favorites /users/:userName/favorites DELETE A JSON object holding the name of the film to be removed
Example:
{
name: "movieName"
} 		A message whether the film has been removed successfully or not from the favorites.
Allow users to deregister by username /users/:userName DELETE The username of the user to be deleted should be put after /users/ in the URL. A verification message if the deregistration has been successful or not.
